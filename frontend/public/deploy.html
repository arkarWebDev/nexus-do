Deploy NexusDo to Production

A step-by-step guide to running NexusDo on a VPS with Docker

Overview

NexusDo is a full-stack task manager: a NestJS backend, Next.js frontend, a Telegram bot, and Neon PostgreSQL — all containerized with Docker Compose and served behind Nginx.

ServicePortWhat it does
Backend (NestJS)3001 (internal)REST API, bot, cron reminders
Frontend (Next.js)3000 (internal)Web dashboard
Nginx80 / 443Reverse proxy + SSL termination

Before You Start

You need a Linux VPS (Ubuntu 22.04 recommended) with at least 1 GB RAM, a domain name pointed to its IP address, and a Neon PostgreSQL database. Sign up at neon.tech (free tier works) and copy your connection string.

1

Install Docker and Docker Compose

sudo apt update && sudo apt install -y docker.io docker-compose-v2
sudo usermod -aG docker $USER
# Log out and back in for group changes to take effect
2

Verify installation

docker --version
docker compose version

Clone and Configure

3

Clone the repository

git clone https://github.com/arkarWebDev/nexus-do.git
cd nexus-do
4

Create your .env file

cp .env.example .env
nano .env

Fill in these values:

VariableWhat to set
DATABASE_URLYour Neon PostgreSQL connection string (from the Neon dashboard)
TELEGRAM_BOT_TOKENYour bot token from @BotFather
CORS_ORIGINYour domain (e.g. https://nexusdo.example.com)
Your Neon connection string looks like: postgresql://user:pass@ep-xxx.us-east-2.aws.neon.tech/db?sslmode=require

Start the Application

5

Build and launch all services

docker compose up -d --build

This builds the backend and frontend images, pulls Nginx, and starts everything. The backend automatically runs database migrations on startup.

6

Check that everything is running

docker compose ps

You should see all 4 services with status Up. Check logs if something is wrong:

docker compose logs backend
docker compose logs frontend
7

Verify the app works

Visit your server's IP in the browser. You should see the NexusDo login page. Click Create account to register and test the full flow.

Enable HTTPS (SSL)

You must have a domain pointing to your VPS IP before this step. DNS changes can take a few minutes to propagate.
8

Get an SSL certificate from Let's Encrypt

docker compose run --rm certbot certonly \
  --webroot --webroot-path=/var/www/certbot \
  --email you@example.com --agree-tos --no-eff-email \
  -d your-domain.com
9

Create the SSL nginx config

Create a new nginx file that includes SSL:

cat > nginx-ssl.conf << 'NGINX'
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate     /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    client_max_body_size 10m;

    location ^~ /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location /api {
        proxy_pass http://backend:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        proxy_pass http://frontend:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
NGINX

Replace your-domain.com with your real domain (6 places in the file above).

10

Update docker-compose.yml to use the SSL config

In docker-compose.yml, change the nginx volume line:

# Change this:
- ./nginx.conf:/etc/nginx/conf.d/default.conf:ro

# To this:
- ./nginx-ssl.conf:/etc/nginx/conf.d/default.conf:ro
11

Apply SSL and restart nginx

docker compose up -d --force-recreate nginx
12

Enable automatic SSL renewal

# Add a cron job to renew every week
echo "0 3 * * 1 cd ~/nexus-do && docker compose run --rm certbot renew && docker compose exec nginx nginx -s reload" | crontab -

Maintenance Commands

View logs

docker compose logs -f backend
docker compose logs -f frontend

Restart a service

docker compose restart backend

Update the app

git pull
docker compose up -d --build

Stop everything

docker compose down
